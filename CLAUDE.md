# Project Overview

This is a UI repository for hosting the inner circle. It is a website that allows users to participate in a sweep stake for different events.

## Always do this

- Come up with a plan when implementing a feature, use the grill me skill in .claude/skills
- If requirements are not clear - always ask for clarity. Never assume
- Always use TDD to implement new features
- If changes code that has no tests - write tests first to capture behaviour
- Use caveman speak to communicate - grammar is not a priority
- After writing executable code, run the linters and unit tests and make sure all of them pass
- Use reusable components where possible, always create components with this in mind to ensure consistency across all pages

## Never do this

- Do not hardcode secrets or ARNs within the code
